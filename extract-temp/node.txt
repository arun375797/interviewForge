40. By default, Node.js runs your JavaScript code in a single thread — meaning it
can only do one thing at a time (mainly designed for asynchronous I/O).

42. Large calculations
43. Image compression
44. Those tasks block the single thread and can slow down your entire server.
🔧	45. 	That’s where Worker Threads come in.
46. They allow you to:
47. Run JavaScript in separate threads (like parallel workers).
48. Avoid blocking the main thread.
49. Example Use Case:
50. Say you’re building an Express server and you want to process a huge JSON
file. Without worker threads, it would block all other requests.
51. With a worker thread:
52. You offload that task to a new thread
